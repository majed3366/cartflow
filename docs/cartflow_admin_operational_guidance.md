# CartFlow — إرشاد تشغيلي للوحة المشرف

## فلسفة الإرشاد التشغيلي

لوحة `/admin/operations` تهدف إلى أن يفهم المشغّل **المعنى** و**الأثر** قبل الغوص في الأرقام. الطبقة الإرشادية (`derive_admin_operational_guidance`) لا تحسب صحة جديدة: تقرأ فقط مخرجات `build_admin_operational_summary_readonly()` وتترجمها إلى:

- أولوية تشغيلية واضحة  
- فقرات تفسيرية قصيرة  
- خطوات مقترحة عندما يكون الملخص يبرر ذلك  

## مبادئ تجربة المشرف

1. **التهدئة أولاً** — تباعد أقسام، عناوين مرقّمة، وشارات لونية هادئة لا صفارات.  
2. **التفسير فوق المقياس** — كل قسم مهم يرافقه جملة أو فقرة تقول «ماذا يعني هذا لوزني؟».  
3. **لا فوضى DevOps** — لا سلاسل زمنية ثقيلة، لا مخططات معقّدة، لا مصطلحات بنية تحتية.  
4. **لا أسرار** — لا يُعرض في الإرشاد إلا ما ورد في الملخص الآمن أصلاً.

## تسلسل الأولوية

تُشتق الأولوية **بشكل حتمي** من حقول موجودة مسبقاً (تدهور الثقة، تصنيف المنصة، أعلام الضغط، وجود تحذيرات):

| مستوى | اسم عربي | متى (مبسّط) |
|--------|-----------|----------------|
| `critical` | خطر تشغيلي | تدهور ثقة التشغيل، أو تصنيف منصة متدهور، أو ضغط انتقال غير متسق |
| `intervention` | يحتاج تدخل | انتباه تشغيلي دقيق، إعداد معطّل، تكرار مزود/حراس تكرار، أو مزود غير جاهز مع تصنيف المزود |
| `follow_up` | يحتاج متابعة | تنبيه تشغيل، انتباه المزود، Sandbox فقط، أي ضغط مُعلَن، أو تحذيرات من الملخص |
| `normal` | طبيعي | خلاف ذلك عندما يكون الملخص هادئاً نسبياً |

الترتيب داخل الدالة **ثابت**: أول قاعدة تطابق تُحدّد المستوى.

## قواعد تجربة هادئة

- **الأهم أولاً**: حالة المنصة → انتباه مهم → مزود وتشغيل → إعداد → اتجاهات → المتاجر.  
- **حالات الصفر**: استبدال «0» الصامتة بنصوص مثل «لا عوائق في المسح» أو «لم يُمسح متجر بعد» حسب السياق.  
- **الشارات**: ألوان زمردية / كهرمانية / برتقالية / وردية للأولوية فقط — بلا وميض.

## مبدأ «التفسير لا المقياس وحده»

الأرقام تبقى في البطاقات، لكن **الجمل الإرشادية** تجيب: هل هذا طبيعي؟ هل يوجد مخاطر؟ هل يحتاج جلسة عمل؟ ذلك يقلّل الحمل المعرفي ويربط اللوحة بقرارات تشغيل SaaS.

## قابلية التوسع

- إضافة جمل جديدة دون تغيير حساب الثقة أو المزود.  
- ربط مستقبلي بتصدير CSV من نفس الملخص دون توسيع نطاق المنطق.

## الملفات ذات الصلة

- `services/cartflow_admin_operational_guidance.py` —اشتقاق النصوص والأولوية.  
- `templates/admin_operations.html` — التسلسل البصري والأقسام.  
- `routes/admin_operations.py` — تمرير `guidance` إلى القالب.
