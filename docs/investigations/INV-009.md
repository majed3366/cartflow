## Observed Symptoms

1. Two WP-5 database fixture tests fail locally:
   - `test_half_open_boundary_rolling` (expects `abandoned_total == 1`, got `0`)
   - `test_sim_sees_may_production_july_zero` (expects May sim cart count `4`, got `0`)
2. Failures occur in `tests/time_authority/test_wp5_dashboard_knowledge_cross_surface.py`.
3. Pure window-equality tests in the same file (no DB seed) pass.
4. Broader Time Authority suite (excluding those DB cases) and WP-6 Brief/Knowledge suites remain green.

**Evidence-only statement:** The instability is in the **WP-5 cross-surface DB fixture / local DB environment path**, not in Daily Brief Time Authority migration (WP-6).

## Root Cause Analysis

*(Pending — Open.)*

Working hypotheses (unproven):

1. Local TestClient / SQLite fixture isolation vs store seed identity.
2. Session/transaction visibility after `_reset()` + `create_all`.
3. Store identity resolve path returning unexpected store for KPI projection.
4. Timestamp timezone naive/aware mismatch on seeded `last_seen_at`.

Status remains **Open**. Not Ready for Fix until RCA isolates fixture vs product predicate.
