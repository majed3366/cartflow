# Direction B — Flow Symbol

**Phase:** Logo exploration v1 (concept only — not final logo)  
**Design school:** Geometric abstraction (curvilinear minimal)  
**Metaphor:** Continuous operational flow with a **decision notch**

---

## Visual description

A **single continuous path** forms a horizontal S-curve (or flattened loop) with one deliberate **angular notch** — the decision point — before the path **closes or caps** at an outcome node. Stroke weight 2px at 32px master; curves are circular arcs, not freehand.

The silhouette suggests **motion through choice**, not a chat send or circular “refresh” arrow. The notch breaks symmetry so the mark is asymmetric and recognizable.

**Master construction:** 32×32 viewBox; path stroke `#2563EB` (`cf-primary`); notch vertex `#6366F1` (`cf-decision`); outcome cap filled `#1E293B` (`cf-node`).

---

## Visual rationale

“Flow” is the brand name anchor — this direction makes **flow literal** without using marketing arrow clichés. The decision notch encodes **Decide** in the Understand → Decide → Act → Recover chain. One stroke = one operational thread, reflecting CartFlow as a **single coherent system** rather than a bundle of tools.

Asymmetric flow distinguishes the mark from generic “circular sync” icons while staying abstract.

## Strengths

| Area | Detail |
|------|--------|
| **Brand name synergy** | “Flow” visible in form; good for marketplace title lockup |
| **Marketing** | Strong hero mark at 64–128px |
| **Recognition** | Asymmetric notch aids recall |
| **Motion story** | Natural path-draw animation for landing |
| **App icon** | Balanced centered composition on soft blue tint |

---

## Weaknesses

| Area | Detail |
|------|--------|
| **Small sizes** | Curves compress; notch may vanish below 20px |
| **Misread risk** | Without wordmark, could suggest “sync” or “loop” |
| **UI reuse** | Less literal match to dashboard table iconography |
| **Single-color print** | Notch accent lost in monochrome unless shape carries it |

## Selection guidance

**Choose Direction B if** the near-term priority is **Zid marketplace launch and landing page** where the mark must carry emotional and name recognition at medium and large sizes.

**Pair with** a simplified **favicon sub-mark** (outcome cap + short path segment) derived from this direction, or adopt Direction A for favicon if B fails at 16px.
